Welcome, {{ display_name }} — let's get you set up

Connect Claude Code on your machine to your team's data

{{ instance_brand }} gives Claude Code on your computer access to your team's curated data, plugins, third-party tools (Asana, Google Workspace, Atlassian), and shared knowledge — so you can ask questions and get answers in plain language, right from your terminal. This page walks you through the one-time setup (~10 minutes); the install script also connects your tools for you, so there's no extra page to visit. Everything it installs lives in your home folder (~/{{ workspace_dir }}) and can be removed in one command.

What leaves your machine. Session telemetry — prompts, tool calls, and tool responses — flows back to the central catalog so the team can analyse failure patterns. Raw data rows you query locally stay on your machine; only the prompt/response transcript travels. Need a session off the record? Toggle Private in Claude Code (by typing /agnes-private) to disable telemetry for that chat — nothing from that session reaches the catalog.

Your first session

What happens when you launch {{ instance_brand }}

A real session, end-to-end. Five beats: launch → pick a project → Claude loads memory → ask anything → close when done. Plugins, data packages, skills, and shared memory quietly do their job inside this loop — you don't have to manage them.

1

Launch your workspace

Open a terminal in your ~/{{ workspace_dir }} folder and type claude. That single command launches Claude Code wired to {{ instance_brand }}.

~/{{ workspace_dir }} — zsh
$ claude
2

{{ instance_brand }} greets you with today's menu

A short status line shows the workspace, recent sync, and pinned skills. You don't have to read it — it's there if something looks off.

claude
Workspace: ~/{{ workspace_dir }} Synced: 2 min ago · 14 tables · 12 plugins Memory: 38 notes loaded >
3

Pick a project — Claude loads its memory

Each project under ~/{{ workspace_dir }}/Projects/ carries its own context. Switching projects swaps in the right data scopes, plugins, and prior solutions automatically.

4

Ask anything — Claude is now scoped to that project

Plain language. Claude knows what tables it can query, which plugins to call, and what prior analysts already figured out.

claude
> how many active subscribers in CZ last week? running: agnes query — subscribers_daily · filtered country=CZ → 24,807 (week of 2026-05-13)
5

When you're done, just close — everything's saved

Session transcripts feed back into the central catalog so future questions don't trip on the same gap. Anything you marked private (/agnes-private) stays off the record.
