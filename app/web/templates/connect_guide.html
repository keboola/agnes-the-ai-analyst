{# ── The one value everyone needs ──────────────────────────────── #} {% call ds.panel(title='The connector URL', tag='section', attrs='aria-label="Connector URL"') %}

Every agent below uses the same native OAuth 2.1 remote MCP connector — no manually-issued token. Paste this URL into your agent, sign in through the browser, and approve the consent screen. Access is scoped to your {{ instance_brand }} permissions: every table and query is filtered by the same RBAC rules that apply elsewhere.

{{ server_url }}/api/mcp/http

The URL must be reachable over HTTPS — remote MCP clients only allow plaintext HTTP for localhost.

{% endcall %} {# ── Claude Desktop / Claude.ai ────────────────────────────────── #} {% call ds.panel(title='Claude Desktop / Claude.ai', tag='section', attrs='aria-label="Connect Claude"') %}
  1. Open Settings → Connectors → Add custom connector.
  2. Paste the connector URL above and click Add.
  3. Claude registers itself automatically and opens a browser window.
  4. Sign in to {{ instance_brand }} with your usual identity provider.
  5. On the consent screen, review the requested scope and click Allow.

The same flow applies to Claude.ai on the web, under Settings → Connectors.

{% endcall %} {# ── Cursor ────────────────────────────────────────────────────── #} {% call ds.panel(title='Cursor', tag='section', attrs='aria-label="Connect Cursor"') %}
  1. Add an MCP server entry pointing at the connector URL. In .cursor/mcp.json (project) or ~/.cursor/mcp.json (global): 
    {
  "mcpServers": {
    "agnes": { "url": "{{ server_url }}/api/mcp/http" }
  }
}
  2. Open Settings → MCP; the agnes server appears.
  3. Click to sign in — the browser opens for the {{ instance_brand }} login + consent.
  4. The tools become available in Agent / Composer.
{% endcall %} {# ── ChatGPT ───────────────────────────────────────────────────── #} {% call ds.panel(title='ChatGPT connectors', tag='section', attrs='aria-label="Connect ChatGPT"') %}
  1. In ChatGPT, open the connector / custom-tool settings and choose to add a remote MCP server.
  2. Paste the connector URL above.
  3. ChatGPT registers itself and opens the browser for sign-in.
  4. Approve the consent screen to finish.

{% endcall %} {# ── Troubleshooting ───────────────────────────────────────────── #} {% call ds.panel(title='Troubleshooting', tag='section', attrs='aria-label="Troubleshooting"') %}
  • “Could not connect” right after adding the URL — the instance must be reachable over HTTPS at the public host.
  • Browser opens but login loops — the identity provider is misconfigured on the instance; ask an admin.
  • Connected, but catalog is empty — your account has no table grants yet; ask an admin for access.
