Switch app-state DB between DuckDB, side-car Postgres (container alongside the app), and a managed cloud Postgres. Forward-only from DuckDB — once an instance is on Postgres, the DuckDB file is the immutable backup, not a writable target. Container↔Cloud transitions are bidirectional. Allowed transitions: DuckDB → side-car, DuckDB → cloud, side-car → cloud, cloud → side-car. There is no transition back to DuckDB.
