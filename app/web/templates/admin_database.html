Allowed transitions

  • duckdb → side-car — first cutover to in-container Postgres.
  • duckdb → cloud — first cutover, skipping side-car.
  • side-car → cloud — graduate from container to managed Postgres.
  • cloud → side-car — move back to container (DR, cost, or retiring the managed instance).
  • No path back to DuckDB. Once on Postgres, the DuckDB file is the recovery backup, not a writable target.
