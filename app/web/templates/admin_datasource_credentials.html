Google Workspace
OAuth desktop app credentials for the GWS data connector
Loading…
Publishable — same value seen by all analysts. Found in the downloaded client_secret_*.json file.
1
Create or pick a GCP project You need roles/editor (or roles covering APIs + OAuth). 
gcloud projects create my-agnes-oauth --set-as-default
↗ Open GCP Console
2
Enable the required APIs 
gcloud services enable \
  drive.googleapis.com \
  gmail.googleapis.com \
  calendar-json.googleapis.com \
  chat.googleapis.com \
  admin.googleapis.com \
  people.googleapis.com
3
Configure the OAuth consent screen Choose Internal if your users are in the same Google Workspace org (no review required). Choose External only if analysts are outside your org. ↗ Open Consent Screen config
4
Create an OAuth client (Desktop app type) Choose Desktop app as the application type. Download the JSON file. ↗ Open Credentials page
5
Copy credentials from the downloaded JSON Open the client_secret_*.json file. Copy client_id into the field above and client_secret into the secret field, then click Save.
